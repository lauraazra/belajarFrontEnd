Rest = aturan aturan untuk membuat web service

memungkinkaan interaksi mesin dengan mesin

Ilustrasinya seperti di restoran

- Rest API = aturan / mendukung
- ada request dan response
- data yang diterima berupa json

Rest API
- Representasional State Transfer
- Gaya Perancangan
- Stateless
- Roy Fielding
- Hampir selalu menggunakan HTTP
- memungkinkan interaksi antar mesin

Bagaimana WEB bekerja 
HTTP => Hypertext Transfer Protocol
kita akses URL = Uniform Resource Locator
- Alamat dari sebuah Resource

kita mengetikan google.com dikirim ke server dan dikembalikan dalam bentuk HTML (karena interaksi manusia dan mesin) ==> disebut get
Kita mengirim request beserta data di dalam form ke sever, dikembalikan dalam bentuk HTML Respon. ==> disebut post

user request data ke server, server request ke rest server, rest sever ngerespon dengan json dan kode status, server ngerespon ke user berupa HTML.
user dan server disebut rest client

Restful
HTTP Method / verbs
- Get => Mengambil data
- Post => Menambah data
- Put => Mengubah Data
- Delete => Menghapus Data
(Put dan delete tidak bisa digunakan browser)

Stateless
- setiap request dilakukan secara terisolaso
- server tidak menyimpan state apapun 
- setiap request harus berisi semua informasi yang dibutuhkan server

resful API
- Menggunakan HTTP Method yang benar
- Endpointnya cukup sampai nama File

JSON => Javascript Object Notation
- bisa dibaca manusia dan mesin
- berisi key dan value
- lebih sederhana dan jelas dibanding XML
- dapat digunakan beragam bahasa dengan cara tersendiri
- menggnakan format object Javascript
- bisa digunakan untuk konfigurasi
- .json
- Douglas Crockford

JSON vs XML
- JSON format Object, XML Format HTML

Object  => property dengan pasangan key dan value
        => di JS bisa bikin fuction atau method, tapi di json ga ada jadi ga boleh

Tipe data dari Value json (BASONN)
- boolean

- object
- number
